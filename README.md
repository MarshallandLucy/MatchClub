# MatchClub｜个性化社团推荐与报名系统

一个面向校园场景的社团推荐网站：https://marshallandlucy.github.io/MatchClub/ （点开直接进入！）
- 学生先填写个人信息与兴趣问卷
- 系统基于匹配算法推荐最适合的社团（含匹配度与推荐理由）
- 学生可查看社团详情并一键提交报名意向
- 可随时查看自己已投递的社团记录

---

## 项目定位

**MatchClub** 旨在解决“社团太多、不知道怎么选”的问题，核心思路是：
1. 结构化采集用户画像（个人信息 + 问卷）
2. 基于多维标签做匹配评分
3. 提供可解释推荐（百分比 + 维度理由）
4. 在同一流程内完成“推荐 → 浏览 → 投递 → 回看”闭环

适用于：新生社团招新、院系活动推荐、兴趣社群导流等场景。

---

## 已实现功能（基于当前代码识别）

### 1) 三步式引导流程
- **步骤1：填写信息**（姓名、班级必填；手机号选填并校验；一句话介绍）
- **步骤2：匹配问卷**（4步引导式问卷，单选+多选）
- **步骤3：社团选择**（智能推荐、分类筛选、查看详情、提交意向）
- 顶部带有清晰的步骤进度指示与返回逻辑

### 2) 智能推荐算法（可解释）
- 按多个维度计算匹配度：兴趣、时间、技能、活动形式、目标
- 支持单选与多选题的差异化计分
- 加入社团活跃度与评分加成
- 输出：
  - 匹配百分比（0-100）
  - 推荐理由（命中的关键维度）
  - 推荐标记（保证 Top3 可推荐）

### 3) 社团浏览与筛选
- 默认提供“**推荐**”分组（优先看高匹配社团）
- 支持按社团类别筛选：全部、学术、文艺、体育、公益、科技、社交
- 社团卡片展示：标签、评分、成员数、招新人数、匹配度进度条

### 4) 社团详情与报名意向提交
- 点击社团可打开详情弹窗，展示：简介、历史、活动、统计信息
- 推荐社团会额外展示匹配分析与理由标签
- 支持“一键发送意向”
- 避免重复投递同一社团
- 若关键信息不完整（姓名/班级），会提示先完善资料

### 5) 已投递记录管理
- 可查看“已投递”列表
- 展示投递时间与匹配分
- 点击记录可回看对应社团详情

### 6) 本地数据持久化（无后端依赖）
- 使用 `localStorage` 保存：
  - 当前步骤
  - 个人资料
  - 问卷答案
  - 当前筛选标签
  - 已投递社团 ID
  - 全部意向投递记录
- 刷新页面后可保留用户状态，方便演示与迭代

### 7) 内置样例数据
- 目前内置 **20 个社团** 的静态数据
- 每个社团包含：主标签、匹配维度值、活动信息、招新信息、评分与活跃度等

## 技术栈

- **前端框架**：React 18 + Vite 5
- **路由**：React Router（HashRouter）
- **UI**：Tailwind CSS + Radix UI（shadcn 风格组件）
- **状态与数据**：React Hooks + `localStorage`
- **通知反馈**：Sonner
- **图标**：lucide-react

> 当前版本已移除 no-code 构建插件依赖，采用标准 Vite 工程结构，便于直接托管到 GitHub 并通过 GitHub Pages 访问。

## 快速开始

### 1. 环境要求
- Node.js 18+
- npm / yarn / pnpm（任选其一）

### 2. 安装依赖

```bash
npm install
```

### 3. 启动开发环境

```bash
npm run dev
```

### 4. 构建生产包

```bash
npm run build
npm run preview
```

### 5. GitHub Pages 部署

仓库已提供 GitHub Actions 工作流：`.github/workflows/deploy-pages.yml`。  
你只需要：
1. 将默认分支设置为 `main`
2. 在 GitHub 仓库 Settings → Pages 中启用 **GitHub Actions** 作为 Source
3. push 到 `main` 后会自动构建并发布静态站点

## 当前版本特点与边界

### 优点
- 产品闭环完整（信息采集→推荐→投递→记录）
- 推荐结果可解释，用户信任感更好
- 本地持久化，演示体验稳定
- UI 交互完整，移动端友好（窄屏布局）

### 当前边界
- 暂无真实后端与数据库（投递记录仅本地可见）
- 暂无登录/权限系统
- 社团数据为静态样例
- 推荐算法为规则打分，尚未引入学习型模型

---

## 下一步优化建议（用于后续迭代）

1. **后端化投递链路**：接入 API + 数据库，支持社团管理员查看投递
2. **账号体系**：学生登录、社团管理员登录、角色权限隔离
3. **问卷升级**：支持题库配置化、权重可调、结果解释模板化
4. **推荐策略升级**：加入协同过滤/行为反馈，支持“点了但没投递”行为加权
5. **运营能力**：加入活动日历、热门社团榜单、搜索排序与曝光策略
6. **数据看板**：转化漏斗（完成问卷率、详情点击率、投递率）
